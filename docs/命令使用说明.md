# 命令使用说明

这份文档介绍当前仓库中可直接使用的主要命令，以及它们在工作流里的作用。

---

## 一、初始化一个 feature

### 命令

```sh
./scripts/init-feature.sh <feature-id> <feature-slug> [title]
```

### 示例

```sh
./scripts/init-feature.sh 001 demo-search "Demo Search"
```

### 作用

- 创建 `features/<feature-id>-<feature-slug>/`
- 初始化标准工件：
  - `00-intake.md`
  - `01-spec.md`
  - `02-plan.md`
  - `06-tasks.md`
  - `07-implementation-log.md`
  - `08-validation.md`
- 初始化 `state.json`

---

## 二、准备进入某个 stage

### 命令

```sh
./scripts/run-stage.sh <feature-dir> <stage>
```

### 示例

```sh
./scripts/run-stage.sh ./features/001-demo-search spec
```

### 作用

- 检查 stage 前置条件
- 更新 `state.json`
- 调用对应 handler
- 为后续执行做准备

### 支持的 stage

- `intake`
- `spec`
- `plan`
- `tasks`
- `implement`
- `validate`

---

## 三、更顺手地执行 stage

### 命令

```sh
./scripts/execute-stage.sh <feature-dir> <stage>
```

### 示例

```sh
./scripts/execute-stage.sh ./features/001-demo-search spec
```

### 作用

这是 `run-stage.sh` 的封装版，会：

- 准备 stage
- 调用 handler
- 输出下一步建议动作

如果你只是想进入某个 stage，通常优先用这个。

---

## 四、记录某个 artifact 已完成

### 命令

```sh
./scripts/complete-artifact.sh <feature-dir> <artifact>
```

### 示例

```sh
./scripts/complete-artifact.sh ./features/001-demo-search 01-spec.md
```

### 作用

- 记录某个工件已经生成完毕
- 更新 `state.json`
- 推导 `next_stage`
- 调用 `after_artifact_write` hook

这个命令通常在：
- 人工写完 artifact 后
- 或 backend consumer 生成完 artifact 后
调用。

---

## 五、生成 stage bundle

### 命令

```sh
./scripts/build-stage-bundle.sh <feature-dir> <stage>
```

### 示例

```sh
./scripts/build-stage-bundle.sh ./features/001-demo-search spec
```

### 作用

生成：

```text
features/<feature>/.runtime/<stage>.bundle.md
```

### bundle 内容包括

- 当前 stage
- 当前状态
- 下一 stage
- Goal
- 要读的 artifacts
- 要写的 artifacts
- Skill prompt 路径
- Subagent prompt 路径
- 建议执行方式

这是 runtime 和 prompt layer 的桥梁。

---

## 六、使用 Claude Code 作为 backend consumer

### 命令

```sh
./scripts/consume-stage-with-claude.sh <feature-dir> <stage>
```

### 示例

```sh
./scripts/consume-stage-with-claude.sh ./features/006-auto-workflow intake
./scripts/consume-stage-with-claude.sh ./features/005-claude-spec-consumer spec
./scripts/consume-stage-with-claude.sh ./features/005-claude-spec-consumer plan
./scripts/consume-stage-with-claude.sh ./features/005-claude-spec-consumer tasks
./scripts/consume-stage-with-claude.sh ./features/005-claude-spec-consumer implement
./scripts/consume-stage-with-claude.sh ./features/005-claude-spec-consumer validate
```

### 当前支持的 stage

- `intake`
- `spec`
- `plan`
- `tasks`
- `implement`（bounded / experimental）
- `validate`

### 作用

- 读取当前 stage 的输入工件
- 读取对应的 Skill prompt / Subagent prompt / template
- 生成 `.runtime/<stage>.claude-prompt.txt`
- 调用本机 `claude` CLI
- 让 Claude Code 直接写出目标 artifact

### 注意

当前这个命令：
- **不会自动执行 `complete-artifact.sh`**
- **不会自动推进到下一 stage**
- `implement` 虽然已接入，但目前仍属于实验性能力

所以标准用法通常是：

```sh
./scripts/execute-stage.sh <feature-dir> <stage>
./scripts/consume-stage-with-claude.sh <feature-dir> <stage>
./scripts/complete-artifact.sh <feature-dir> <artifact>
```

## 七、查看某个 feature 的状态摘要

### 命令

```sh
./scripts/feature-summary.sh <feature-dir>
```

### 示例

```sh
./scripts/feature-summary.sh ./features/005-claude-spec-consumer
```

### 作用

输出一个简洁摘要，包括：

- 当前路径
- current stage
- status
- last artifact
- 核心工件是否存在

适合快速看某条 feature 走到了哪里。

---

## 八、推荐的手动流程

当前系统最推荐的手动流程是：

### 1. 初始化 feature

```sh
./scripts/init-feature.sh 006 my-feature "My Feature"
```

### 2. 进入某个阶段

```sh
./scripts/execute-stage.sh ./features/006-my-feature intake
```

### 3. 写工件或调用 backend consumer

例如在 `spec` 阶段：

```sh
./scripts/consume-stage-with-claude.sh ./features/006-my-feature spec
```

### 4. 完成工件后记录状态

```sh
./scripts/complete-artifact.sh ./features/006-my-feature 01-spec.md
```

### 5. 继续下一个 stage

```sh
./scripts/execute-stage.sh ./features/006-my-feature plan
```

## 九、自动 workflow

### 命令

```sh
./scripts/auto-workflow.sh <feature-dir> <mode> [args...]
```

### 支持的 mode

#### `full`
执行一个连续的完整流程。

```sh
./scripts/auto-workflow.sh ./features/006-auto-workflow full intake tasks
```

默认值：
- start stage = `intake`
- end stage = `tasks`

#### `single`
只执行单个阶段。

```sh
./scripts/auto-workflow.sh ./features/005-claude-spec-consumer single validate
```

#### `range`
执行一个阶段区间。

```sh
./scripts/auto-workflow.sh ./features/005-claude-spec-consumer range plan tasks
```

### 当前自动化覆盖范围

当前可自动消费的阶段：
- `intake`
- `spec`
- `plan`
- `tasks`
- `implement`（bounded / experimental）
- `validate`

说明：
- `implement` 已接入自动 consumer，但目前仍属于实验性能力
- 更适合受控、边界明确的实现任务

### 当前执行逻辑

`auto-workflow.sh` 对每个阶段会做：

1. `execute-stage.sh`
2. 调用 backend consumer（如果该阶段已支持）
3. `complete-artifact.sh`
4. 如果是 `validate`，再调用 `after_validation.sh`
5. 如果是 `implement`，会先经过风险评估和 checkpoint 判定

这意味着它已经支持：
- 整体自动执行一个早期流程
- 或只执行一个特定阶段
- 或执行一个阶段区间
- 并对 `implement` 使用受控自动化策略

---

## 十、当前命令层的限制

当前命令层还没有做到：

- 真正成熟稳定的 implement 全自动执行
- 自动从失败中恢复
- 多 backend 自由切换
- 完整无人工审批的生产级工作流

也就是说，目前已经是：

> **早期自动化 + implement 实验性接线**

而不是彻底成熟的全自动工程平台。

---

## 十一、一句话理解各命令职责

你可以这样记：

- `init-feature.sh` → 创建一条 feature
- `run-stage.sh` → 进入某个阶段
- `execute-stage.sh` → 更顺手地进入阶段
- `build-stage-bundle.sh` → 生成执行包
- `consume-stage-with-claude.sh` → 让 Claude 跑某个阶段
- `complete-artifact.sh` → 记录工件完成
- `feature-summary.sh` → 看状态
- `auto-workflow.sh` → 自动跑一个阶段 / 阶段区间 / 一段完整流程

---

## 十二、推荐组合

如果你现在要手动跑一个阶段，最常见组合就是：

```sh
./scripts/execute-stage.sh ./features/<feature> <stage>
./scripts/consume-stage-with-claude.sh ./features/<feature> <stage>
./scripts/complete-artifact.sh ./features/<feature> <artifact>
```

如果你现在要自动跑一个流程，则优先用：

```sh
./scripts/auto-workflow.sh ./features/<feature> full intake tasks
```

这是当前最标准的两类使用方式。
