# 轻量级v1

## 1. 概述

本文档定义了SDD流水线的轻量级v1实现。这是一个最小可行的产品，专注于核心功能而不过度工程。

---

## 2. 设计原则

### 2.1 最小化

只构建必要的：
- 核心阶段
- 基础制品
- 关键hooks
- 简单状态管理

### 2.2 实用性

优先考虑：
- 可用性 > 完整性
- 简单 > 复杂
- 迭代 > 完美

### 2.3 可扩展性

设计允许：
- 轻松添加新阶段
- 定制hooks
- 集成新后端

---

## 3. 核心组件

### 3.1 阶段 (6个)

v1包含6个核心阶段：

1. **intake** - 需求收集
2. **spec** - 规格编写
3. **plan** - 计划制定
4. **tasks** - 任务分解
5. **implement** - 代码实现
6. **validate** - 验证确认

### 3.2 制品 (6个)

必需的制品：

- `00-intake.md`
- `01-spec.md`
- `02-plan.md`
- `06-tasks.md`
- `07-implementation-log.md`
- `08-validation.md`

可选的制品：
- `03-research.md`
- `04-data-model.md`
- `05-contracts.md`
- `09-retrospective.md`

### 3.3 Skills (6个)

每个阶段一个Skill：
- `sdd-intake`
- `sdd-spec`
- `sdd-plan`
- `sdd-tasks`
- `sdd-implement`
- `sdd-validate`

### 3.4 Subagents (6个)

每个阶段一个角色：
- intake-writer
- spec-writer
- planner
- task-decomposer
- implementer
- validator

---

## 4. 技术实现

### 4.1 技术栈

- **Shell脚本** - 编排逻辑
- **Markdown** - 制品格式
- **JSON** - 状态存储
- **Claude Code** - 后端消费者

### 4.2 目录结构

```
.
├── scripts/          # 编排脚本
├── runtime/handlers/ # 阶段处理器
├── hooks/            # Hook脚本
├── skills/           # Skill提示词
├── subagents/        # 子代理提示词
├── templates/        # 制品模板
├── features/         # Feature文件夹
└── docs/             # 文档
```

---

## 5. 状态管理

### 5.1 state.json

每个feature有一个简单的状态文件：

```json
{
  "feature_id": "001",
  "feature_slug": "my-feature",
  "feature_title": "My Feature",
  "current_stage": "spec",
  "status": "in_progress",
  "last_artifact": "00-intake.md",
  "next_stage": "plan"
}
```

### 5.2 状态转换

```
intake -> spec -> plan -> tasks -> implement -> validate -> done
```

---

## 6. 自动化

### 6.1 支持的模式

- **手动** - 每步手动执行
- **单阶段** - 运行单个阶段
- **阶段范围** - 运行阶段范围
- **完整** - 运行完整流程（部分自动化）

### 6.2 当前限制

- implement阶段是实验性的
- 不支持完整自动化（需要人工审批）

---

## 7. 扩展点

### 7.1 可选的增强

v1之后可以添加：
- 更多hooks
- 更多制品
- 回顾阶段
- 高级验证
- 多个后端

### 8.1 核心指标

- 可执行的完整流程
- 制品质量
- 开发者满意度

### 8.2 非目标

v1不追求：
- 完美的代码质量
- 完整的功能覆盖
- 企业级特性

## 10. 开始使用

### 10.1 快速开始

```bash
# 初始化feature
./scripts/init-feature.sh 001 my-feature "My Feature"

# 运行完整流程
./scripts/auto-workflow.sh ./features/001-my-feature full intake tasks
```

### 10.2 手动流程

```bash
# 初始化
./scripts/init-feature.sh 001 my-feature "My Feature"

# 需求收集
./scripts/execute-stage.sh ./features/001-my-feature intake
# ... 编写 intake ...
./scripts/complete-artifact.sh ./features/001-my-feature 00-intake.md

# 规格
./scripts/execute-stage.sh ./features/001-my-feature spec
./scripts/consume-stage-with-claude.sh ./features/001-my-feature spec
./scripts/complete-artifact.sh ./features/001-my-feature 01-spec.md

# ... 继续其他阶段 ...
```

---

## 11. 限制

v1的已知限制：

1. 不支持复杂的依赖关系
2. 没有内置的版本控制
3. 验证是基础的
4. 需要人工干预（有意为之）

---

## 12. 总结

轻量级v1是一个最小可行的SDD流水线实现。它提供了：
- 核心的阶段流程
- 基础的制品管理
- 简单的状态跟踪
- 基本的自动化

它的设计允许随着经验积累而轻松扩展。
