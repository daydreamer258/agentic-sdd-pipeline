# 后端消费者

## 1. 概述

本文档定义了后端消费者 - 负责实际执行阶段工作的AI后端集成。

---

## 2. 什么是后端消费者

后端消费者是：
- 实际调用AI模型执行工作的组件
- 编排脚本和AI之间的桥梁
- 将提示词转换为实际输出的引擎

---

## 3. 支持的后端

### 3.1 Claude Code

**状态**: 完全支持

**阶段支持**:
- intake ✅
- spec ✅
- plan ✅
- tasks ✅
- implement ✅ (实验性)
- validate ✅

**配置**:
- 使用 `claude` CLI
- 需要本地安装Claude Code

---

## 4. Claude Code集成

### 4.1 集成架构

```
scripts/consume-stage-with-claude.sh
           |
           v
    构建完整提示词
    (Skill + Subagent + 模板 + 上下文)
           |
           v
      调用Claude CLI
           |
           v
      写出制品文件
           |
           v
      返回结果
```

### 4.2 提示词构建

消费者构建提示词的方式：

```bash
# 1. 读取Skill
SKILL=$(cat skills/sdd-$STAGE/SKILL.md)

# 2. 读取Subagent
SUBAGENT=$(cat subagents/$SUBAGENT_NAME.md)

# 3. 读取模板
TEMPLATE=$(cat templates/$ARTIFACT.md)

# 4. 读取输入制品
INPUTS=$(cat $INPUT_PATHS)

# 5. 组合
FULL_PROMPT="$SKILL\n\n$SUBAGENT\n\n$TEMPLATE\n\n## Context\n$INPUTS"
```

### 4.3 使用示例

```bash
# 运行spec阶段
./scripts/consume-stage-with-claude.sh ./features/001-my-feature spec

# 运行plan阶段
./scripts/consume-stage-with-claude.sh ./features/001-my-feature plan

# 运行tasks阶段
./scripts/consume-stage-with-claude.sh ./features/001-my-feature tasks
```

## 5. 消费流程

### 5.1 步骤

1. **验证输入** - 检查必需的制品存在
2. **构建提示词** - 组合所有部分
3. **调用后端** - 执行AI调用
4. **处理输出** - 写入制品文件
5. **报告结果** - 返回成功/失败

### 5.2 错误处理

- 验证失败：报告错误，停止
- 构建失败：报告错误，停止
- 调用失败：重试（可选），报告错误
- 输出失败：报告错误，保留临时文件

## 6. 添加新的后端

### 6.1 接口要求

新的消费者脚本必须：

```bash
# 接受两个参数：
# $1: feature目录
# $2: 阶段名称

# 成功时：
# - 写出制品
# - 返回0

# 失败时：
# - 返回非0
# - 报告错误
```

### 6.2 示例模板

```bash
#!/usr/bin/env sh
FEATURE_DIR="$1"
STAGE="$2"

# 1. 验证输入
if [ ! -d "$FEATURE_DIR" ]; then
    echo "Error: Feature directory not found"
    exit 1
fi

# 2. 构建提示词
# ... 构建逻辑 ...

# 3. 调用后端
# ... 调用逻辑 ...

# 4. 处理输出
# ... 写入制品 ...

exit 0
```

### 6.3 集成到工作流

要集成新后端：

1. 创建消费者脚本
2. 更新 `auto-workflow.sh` 支持新后端
3. 测试各阶段
4. 更新文档

## 9. 限制

### 9.1 当前限制

- 仅支持Claude Code
- 不支持多后端切换
- 无本地缓存

### 9.2 已知问题

- 大型制品可能需要分块处理
- 复杂规格可能需要多次迭代

---

## 10. 故障排除

### 10.1 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| CLI未找到 | 未安装Claude | 安装Claude Code |
| 权限错误 | 无写入权限 | 检查目录权限 |
| 超时 | 请求太复杂 | 简化提示词 |

### 10.2 调试步骤

1. 检查Claude是否安装
2. 验证目录权限
3. 尝试手动运行提示词
4. 检查日志输出
