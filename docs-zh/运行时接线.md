# 运行时接线

## 1. 概述

本文档描述了流水线的运行时接线机制 - 如何将各组件连接在一起工作。

---

## 2. 架构概述

```
┌─────────────────────────────────────────────────────────────┐
│                      编排脚本层                              │
│  (run-stage.sh, execute-stage.sh, auto-workflow.sh)         │
└─────────────────────────┬───────────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────────┐
│                      Hook层                                 │
│  (before_stage_transition, after_artifact_write, etc.)      │
└─────────────────────────┬───────────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────────┐
│                      运行时处理器                           │
│  (runtime/handlers/<stage>.sh)                              │
└─────────────────────────┬───────────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────────┐
│                      制品存储                               │
│  (features/<id>-<slug>/)                                    │
└─────────────────────────────────────────────────────────────┘
```

---

## 3. 核心组件

### 3.1 编排脚本

| 脚本 | 职责 |
|------|------|
| `init-feature.sh` | 创建新的feature文件夹 |
| `run-stage.sh` | 运行单个阶段（准备+执行） |
| `execute-stage.sh` | 执行阶段的便捷包装 |
| `auto-workflow.sh` | 自动运行多个阶段 |
| `consume-stage-with-claude.sh` | 调用Claude Code后端 |
| `complete-artifact.sh` | 标记制品完成并更新状态 |

### 3.2 Hooks

| Hook | 触发时机 | 用途 |
|------|----------|------|
| `on_feature_init` | Feature初始化时 | 初始化scaffolding |
| `before_stage_transition` | 进入新阶段前 | 验证前置条件 |
| `before_implement` | 实现阶段开始前 | 风险评估 |
| `after_artifact_write` | 制品写入后 | 后续处理 |
| `after_validation` | 验证完成后 | 最终处理 |

### 3.3 运行时处理器

每个阶段有一个处理器：
- `runtime/handlers/intake.sh`
- `runtime/handlers/spec.sh`
- `runtime/handlers/plan.sh`
- `runtime/handlers/tasks.sh`
- `runtime/handlers/implement.sh`
- `runtime/handlers/validate.sh`

每个处理器：
1. 验证前置条件
2. 调用 `build-stage-bundle.sh` 生成运行时包
3. 报告准备就绪

---

## 4. 阶段包 (Stage Bundle)

`build-stage-bundle.sh` 为每个阶段生成一个运行时包。

### 包内容

```
.runtime/<stage>.bundle.md
```

包含：
- 当前阶段信息
- 目标
- 要读取的制品
- 要写的制品
- Skill路径
- Subagent路径
- 执行建议

### 包生成过程

```sh
./scripts/build-stage-bundle.sh <feature-dir> <stage>
```

## 5. 状态管理

### 5.1 state.json

每个feature有一个 `state.json`：

```json
{
  "feature_id": "001",
  "feature_slug": "my-feature",
  "feature_title": "My Feature",
  "current_stage": "spec",
  "status": "in_progress",
  "last_artifact": "00-intake.md",
  "next_stage": "plan",
  "last_updated_at": "2026-03-24T10:00:00Z",
  "validation_result": "",
  "needs_review": false
}
```

### 5.2 状态库

`scripts/state-lib.sh` 提供状态操作函数：

- `state_get` - 读取状态值
- `state_write` - 写入状态
- `state_now` - 获取当前时间戳
- `next_stage_for_artifact` - 推导下一阶段

## 6. 流程示例

### 手动流程

```sh
# 1. 初始化feature
./scripts/init-feature.sh 001 my-feature "My Feature"

# 2. 进入spec阶段
./scripts/execute-stage.sh ./features/001-my-feature spec

# 3. 运行Claude生成规格
./scripts/consume-stage-with-claude.sh ./features/001-my-feature spec

# 4. 标记完成
./scripts/complete-artifact.sh ./features/001-my-feature 01-spec.md

# 5. 进入plan阶段
./scripts/execute-stage.sh ./features/001-my-feature plan
```

### 自动流程

```sh
# 运行完整早期阶段
./scripts/auto-workflow.sh ./features/001-my-feature full intake tasks
```

## 7. 后端消费者集成

### 7.1 Claude Code集成

`consume-stage-with-claude.sh` 调用Claude Code：

1. 加载Skill提示词
2. 加载Subagent提示词
3. 加载模板
4. 读取输入制品
5. 生成完整提示词
6. 调用Claude CLI
7. 写出制品

### 7.2 支持的阶段

- intake ✅
- spec ✅
- plan ✅
- tasks ✅
- implement ✅ (实验性)
- validate ✅

---

## 8. 错误处理

### 8.1 前置条件检查

在进入阶段前验证：
- 必需的制品是否存在
- 上一阶段是否完成

### 8.2 阶段内错误

如果阶段执行失败：
- 报告错误
- 不更新状态
- 允许重试

### 8.3 实现阶段特殊处理

实现阶段有额外的安全检查：
- 风险评估
- 检查点生成（如果需要审批）

---

## 9. 配置点

### 9.1 Hooks

可以在以下位置添加自定义hook：
- `hooks/before_stage_transition.sh`
- `hooks/after_artifact_write.sh`
- 等

### 9.2 处理器

可以通过修改 `runtime/handlers/<stage>.sh` 自定义阶段行为。

### 9.3 提示词

可以通过编辑以下文件自定义提示词：
- `skills/sdd-*/SKILL.md`
- `subagents/*.md`

## 11. 扩展点

### 添加新阶段

1. 创建 `runtime/handlers/<new-stage>.sh`
2. 创建 `skills/sdd-<new-stage>/SKILL.md`
3. 创建对应的模板文件
4. 更新 `state-lib.sh` 中的阶段映射
5. 添加必要的hooks

### 添加新后端

1. 创建 `scripts/consume-stage-with-<backend>.sh`
2. 实现消费逻辑
3. 集成到 `auto-workflow.sh`
